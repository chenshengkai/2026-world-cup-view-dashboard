# 2026 世界杯观赛看板

日历风格的世界杯赛程看板。直接用浏览器打开 `index.html`，或运行本地服务：

```bash
python3 -m http.server 8123   # 然后访问 http://localhost:8123
```

## 功能

- **日历视图**：2026-06-11 至决赛，按你所在时区（如北京时间）归档日期、显示开球时间；今天高亮
- **比赛卡片**：阶段标签（小组赛→决赛按色区分）、对阵双方国旗+中文队名、比分（完赛后）或开球时间；悬停卡片显示球场/城市/当地开球时间
- **球队弹窗**：鼠标悬停（或点击固定）任一队名，显示队徽、主教练、捧杯次数与年份、历史最佳战绩、按位置分组的26人名单（姓名/年龄/位置）
- **赛况弹窗**：鼠标悬停比分，按主客队左右分栏显示进球（球员/助攻/时间，含点球、乌龙标注）和红黄牌；数据存于 schedule JSON 的可选 `events` 字段：`{"home": [{"type": "goal|pen|og|yellow|red", "minute": "45+2", "player": "...", "assist": "..."}], "away": [...]}`
- **分组积分榜**（`standings.html`）：顶部"📊 分组积分榜"链接进入。12 个小组积分表（赛/胜/平/负/进/失/净/分），按 FIFA 规则排名（积分→净胜球→进球→相互战绩），前两名绿色、第 3 名橙色标注；另附"最佳第三名"排名（前 8 名晋级）。完全由 `data.js` 客户端实时计算，比分刷新后自动更新，无需额外数据步骤。
- **关注球队**：顶部下拉选择后高亮该队全部比赛
- 淘汰赛未定对阵显示中文占位符（如"A组第1"、"第73场胜者"）

## 文件结构

| 文件 | 说明 |
|---|---|
| `index.html` / `app.js` | 赛程看板页（日历视图），无构建依赖 |
| `standings.html` / `standings.js` | 分组积分榜页，从 `data.js` 客户端实时计算排名 |
| `popup.js` | 球队信息弹窗，两个页面共用 |
| `styles.css` | 两个页面共用的样式 |
| `data.js` | 前端使用的合并数据（由脚本生成，勿手改） |
| `data/raw/schedule/*.json` | 5个赛程分片（小组赛×3、1/16决赛、1/8决赛至决赛） |
| `data/raw/teams/*.json` | 48支球队资料（教练、26人名单、历史战绩、队徽URL） |
| `build_data.py` | 合并 raw 数据 → `data.js`，含完整性校验和城市名标准化 |

## 更新比分 / 刷新数据

**自动刷新**：已配置每天 15:30 运行的定时任务 `wc2026-daily-score-refresh`（Claude 桌面应用「Scheduled」侧栏可管理）。它会抓取前一比赛日的全部赛果写入 `data/raw/schedule/`、替换已确定的淘汰赛对阵，并重跑 `build_data.py`。应用关闭时任务顺延到下次启动时执行。

**手动刷新**：

1. 编辑 `data/raw/schedule/` 对应分片，为完赛比赛补上 `home_score`、`away_score` 并把 `status` 改为 `finished`；点球大战加 `"penalties": "4:2"`（主队在前）；淘汰赛对阵确定后把占位符替换为球队英文名，需与 `data/raw/teams/*.json` 的 `en` 一致
2. 重新运行 `python3 build_data.py`（校验失败会列出具体问题并退出码1）

数据来源：英文维基百科 / FIFA / ESPN 多源交叉核对（35个研究agent + 5个独立核对agent）。
